MoE TP Forward Pass Design

Replicated Routing with Distributed Experts

The Algorithm

  1. Replicated Routing: Every rank runs self.router(x). Since weights are identical and input is replicated, every rank gets the same indices and gates.
  2. Expert Computation: For each top-k slot:
    • Identify which expert is needed for each token.
    • Pass tokens through self.experts[idx].
    • Crucial: ShardedExpert (via ShardedLinear) will handle the MPI communication to ensure every rank gets the full output of that expert.
  3. Gated Accumulation: Multiply expert outputs by their corresponding gates and sum them into the final outputs array.
TP Forward Flow (Per Rank)
Input X (Full) → [Router] → Indices/Gates (Full)

Loop k in top-k:
  Loop i in batch:
    e = Indices[i, k]
    out = Experts[e](X[i]) ← Collective happens inside here!
    Outputs[i] += Gates[i, k] * out

Implementation Note

While the loop-based approach (token-by-token) is easy to write, in a real system we'd group tokens assigned to the same expert to use faster matrix-matrix multiplications. For this assignment, the loop-based approach is perfectly fine to start!
